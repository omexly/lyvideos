# 🌐 OmeTV Clone Premium - دليل التشغيل والرفع للاستضافة

هذا المشروع عبارة عن تطبيق ويب لمحادثات الفيديو العشوائية شبيه بـ OmeTV، مبرمج باستخدام **Node.js, Express, Socket.io, WebRTC** ويحتوي على ميزات VIP وفلترة متقدمة للبلدان والجنس بالإضافة إلى لوحة تحكم إدارية كاملة للآدمن.

---

## 🚀 التشغيل المحلي (Local Running)

تم تصميم النظام ليعمل فوراً وبشكل تلقائي محلياً دون الحاجة لتثبيت أي برامج لقاعدة البيانات (حيث يقوم باستخدام قاعدة بيانات محلية باسم `local_db.json` بشكل احتياطي إذا لم تتوفر إعدادات MongoDB).

### 1. تثبيت الاعتمادات
تأكد من وجود Node.js على جهازك، ثم افتح منفذ الأوامر (CMD) في مجلد المشروع وقم بتشغيل:
```bash
npm install
```

### 2. تشغيل المشروع
لتشغيل خادم التطبيق محلياً:
```bash
npm start
```
أو للتشغيل في بيئة التطوير (إعادة تشغيل تلقائي عند التعديل):
```bash
npm run dev
```

سيتم تشغيل الموقع على الرابط التالي: [http://localhost:5000](http://localhost:5000)

### 🔑 حساب الآدمن الافتراضي (للتجربة)
عند بدء تشغيل النظام لأول مرة محلياً، سيتم إنشاء حساب آدمن تلقائي للتجربة:
- **اسم المستخدم (Username):** `admin`
- **كلمة المرور (Password):** `admin123`

## 👑 تجربة ميزات الـ VIP والفلترة
لتجربة الفلاتر الخاصة بالـ VIP محلياً:
1. قم بإنشاء حساب جديد من زر (تسجيل الدخول / حساب جديد).
2. اضغط على زر **"تفعيل VIP مجاناً للتجربة"** في القائمة الجانبية.
3. سيتم ترقية حسابك فوراً إلى VIP وستفتح لك خيارات فلترة الجنس والدولة.

## 🗄️ إعداد قاعدة البيانات السحابية (MongoDB Atlas) - مجاناً مدى الحياة

عند رفع المشروع للاستضافة، يجب استخدام قاعدة بيانات سحابية لأن الاستضافة المجانية تمسح الملفات المحلية (مثل `local_db.json`) عند إعادة التشغيل.

1. توجه إلى [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) وأنشئ حساباً مجانياً.
2. قم بإنشاء **Cluster** جديد مجاني (Shared Cluster).
3. اضغط على **Connect** ثم اختر **Drivers** للحصول على رابط الاتصال.
4. انسخ الرابط الذي يشبه التالي:
   `mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority`
5. استبدل `<username>` و `<password>` ببيانات مستخدم قاعدة البيانات الذي أنشأته.
6. ضع هذا الرابط في ملف `.env` تحت متغير `MONGODB_URI` للتشغيل المحلي السحابي، أو في إعدادات الاستضافة.

## ☁️ الرفع على استضافة مجانية (Render.com)

Render هي من أفضل الاستضافات المجانية التي تدعم الـ WebSockets وتوفر تشغيل تطبيقات Node.js.

### 1. رفع الكود على GitHub
- أنشئ مستودعاً جديداً على حسابك في GitHub (مثال: `ometv-clone`).
- ارفع ملفات هذا المشروع إلى المستودع.
- *ملاحظة*: تأكد من عدم رفع ملف `.env` أو `node_modules` (تلقائياً مستثناة عبر `.gitignore` إذا وُجد).

### 2. إعداد الحساب في Render.com
- توجه إلى [Render.com](https://render.com) وسجل دخولك باستخدام حساب GitHub الخاص بك.
- اضغط على **New** ثم اختر **Web Service**.
- قم بربط مستودع الـ GitHub الخاص بالمشروع.

### 3. إعدادات البناء والتشغيل
- **Name:** اكتب اسماً للمشروع (مثال: `my-ometv-app`).
- **Runtime:** `Node`
- **Build Command:** `npm install`
- **Start Command:** `node server.js`

### 4. إضافة متغيرات البيئة (Environment Variables)
في نفس صفحة الإعدادات، انتقل إلى تبويب **Environment** وأضف المتغيرات التالية:
1. `JWT_SECRET` : ضع أي نص عشوائي طويل (لحماية كلمات المرور والتوكنز).
2. `MONGODB_URI` : ضع رابط اتصال قاعدة بيانات MongoDB Atlas السحابية الذي حصلت عليه في الخطوة السابقة.

اضغط على **Create Web Service**. سيقوم الموقع ببناء التطبيق وتشغيله خلال دقائق وسيعطيك رابطاً مجانياً يعمل بـ HTTPS (وهو ضروري جداً لتعمل الكاميرا والميكروفون WebRTC بشكل آمن!).

## 🔒 ملاحظة هامة جداً بخصوص الكاميرا (WebRTC)
تفرض المتصفحات الحديثة (Chrome, Safari, Firefox) قيود أمان صارمة؛ **لن تعمل الكاميرا أو الميكروفون** إلا في الحالات التالية:
1. التشغيل المحلي على `localhost` (مثال: `http://localhost:5000`).
2. الروابط التي تستخدم بروتوكول آمن مشفر **HTTPS** (والذي توفره استضافة Render تلقائياً بشكل مجاني).
3. لن تعمل الكاميرا إذا قمت بفتح الموقع محلياً عن طريق رقم الـ IP الخاص بجهازك بدون بروتوكول HTTPS.
